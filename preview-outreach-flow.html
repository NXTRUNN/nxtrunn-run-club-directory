Bottleneck Analysis

1. Verification Code Email is Plain Text

The outreach email is a beautiful branded HTML email. But when they actually click "Claim" and enter their email, the verification code arrives as plain text with no branding. This inconsistency can make it look like a phishing attempt or spam. It also has no "From" name — it uses the default WordPress sender.

Fix: Make the verification code email branded HTML with the same look as outreach. Add From: NXTRUNN Run Clubs <runclubs@nxtrunn.com>

2. From Address Mismatch

Outreach emails currently send from treb@nxtrunn.com. The verification code email sends from WordPress default (likely wordpress@nxtrunn.com or info@). The user created runclubs@nxtrunn.com for all directory mail. All 3 emails should use the same sender.

Fix: Set all emails to From: NXTRUNN Run Clubs <runclubs@nxtrunn.com>

3. Registration Wall (Step 4)

Club owners clicking "Claim Your Club" from the email are likely first-time visitors. If they're not logged in, they hit a MemberPress registration page. This is the biggest drop-off point — they came to claim a club, not create an account. The redirect back to the directory after signup relies on a cookie, which may not survive cross-domain or incognito browsing.

Consider: Add copy above "Sign Up to Claim" that says "Create a free account to verify your club (takes 30 seconds)"

4. Two Separate Email Steps

The user gets the outreach email, clicks through, signs up, then has to enter their club email AGAIN in the claim form, then check that inbox for a code. That's 2 email checks + a signup. Each handoff is a drop-off point.

Consider: Pre-fill the club email in the claim form if the outreach email matches the contact email on file

5. No "Claim" CTA on the Club Card Itself

If someone lands on the directory (not via the email link), they have to: find their club → click it → scroll to the bottom of the modal → find "Claim This Club." There's no visual indicator on the card that it's unclaimed or claimable.

Consider: Show a subtle "Claim" link or unclaimed indicator on cards for logged-in users

6. No Confirmation Email After Successful Claim

After verifying the code, the owner sees a success message in the modal — but gets no email confirmation. A "Welcome, you now own [Club Name]" email would reinforce the action and remind them to edit their listing.

Consider: Send a branded "You're verified!" confirmation email with a direct link to edit
